01

What It Is

Research spins up a background agent to investigate a question against primary sources, then captures the findings as a cited Markdown file in your repo.

It is legwork you delegate, not thinking you outsource. You keep your conversation going while it reads. When it's done, you get a document to react to — with every claim traced to its source.

  • Input: a question or investigation topic
  • Output: a single Markdown file with inline citations
  • Constraint: primary sources only
Investigate against primary sources — official docs, source code, specs, first-party APIs — not a secondary write-up of them.

The single constraint of the Research skill

Every claim must follow back to the source that owns it. No summaries of summaries.

02

The Background Agent Pattern

The defining architectural move: reading happens off-thread.

You invoke /research with a question. A background agent spawns. Your main conversation continues uninterrupted. The agent reads, verifies, writes — and drops the result file when done.

This is not outsourcing. It's parallelization of the boring part.

04

Research vs Other Skills

Research

Find out — read primary sources, gather facts, produce cited document. Background agent. Upstream of thinking.

Grilling

Decide — interview the user about a plan or design. HITL conversation. One question at a time. Sharpening, not gathering.

Prototype

Explore — cheap, rough artifacts to react to. Throwaway code that reveals "how it should look/behave."

Wayfinder

Map — chart investigation tickets. Research tickets are one type. Wayfinder plans; Research fills in the blanks.

07

Research as a Wayfinder Ticket Type

In the Wayfinder planning system, research is one of four ticket types. It's the AFK (Away From Keyboard) type — fully autonomous, no human interaction needed.

  • Research — AFK: read docs, APIs, code → Markdown summary
  • Prototype — HITL: build cheap artifacts → human reacts
  • Grilling — HITL: interview → human decides
  • Task — HITL or AFK: do manual work → result recorded

Research is the fuel for Wayfinder maps. You can't chart what you haven't researched.

09

When to Reach for It

  • Next step is finding something out, not deciding
  • API behavior question: "What does this endpoint actually return?"
  • Claim verification: "Does the spec say X?"
  • Reading legwork: "Gather all the auth requirements"
  • Upstream of thinking: research feeds grilling, planning, design

Don't use when: You need to sharpen a plan by interview (use Grilling) or explore what to build with code (use Prototype).
